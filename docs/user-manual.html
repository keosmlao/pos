1. ພາບລວມລະບົບ

ລະບົບ POS ນີ້ອອກແບບໃຫ້ຮ້ານຄ້າໃຊ້ງານຂາຍໜ້າຮ້ານ ແລະ ບໍລິຫານຫຼັງຮ້ານໃນລະບົບດຽວ. ໜ້າ POS ໃຊ້ສຳລັບການຂາຍ, ຄິດເງິນ, ອອກໃບບິນ, ຮັບຄືນ ແລະ ສົ່ງເງິນ. Admin Console ໃຊ້ສຳລັບຈັດການສິນຄ້າ, ຊື້ເຂົ້າ, ສາຂາ, ລາຍງານ ແລະ ການຕັ້ງຄ່າ.

2. ການໃຊ້ງານໜ້າ POS

ຂັ້ນຕອນຂາຍສິນຄ້າປົກກະຕິ

  1. ເຂົ້າລະບົບ ແລ້ວຢູ່ໜ້າ POS.
  2. ເລືອກສາຂາທີ່ຈະຂາຍ ຖ້າລະບົບມີຫຼາຍສາຂາ.
  3. ສະແກນ barcode ຫຼື ພິມລະຫັດ/ຊື່ສິນຄ້າໃນຊ່ອງຄົ້ນຫາ.
  4. ຖ້າສິນຄ້າບໍ່ມີ barcode ໃຫ້ກົດ Catalog ເພື່ອເລືອກຈາກໝວດໝູ່.
  5. ປັບຈຳນວນດ້ວຍປຸ່ມ + ຫຼື - ໃນລາຍການບິນ.
  6. ເລືອກສະມາຊິກ ຫຼື ປ່ອຍເປັນລູກຄ້າທົ່ວໄປ.
  7. ກວດຍອດລວມ ແລະ ກົດ ຊຳລະ.

ການຈັດການລາຍການໃນບິນ

ຫຼຸດຈຳນວນກົດ - ຂ້າງສິນຄ້າ. ຖ້າຈຳນວນເປັນ 0 ລາຍການຈະຖືກລຶບ.
ລຶບສິນຄ້າກົດປຸ່ມລຶບໃນແຖວສິນຄ້າ.
ລ້າງບິນກົດ ລ້າງທັງໝົດ ເມື່ອຕ້ອງການເລີ່ມບິນໃໝ່.

ສະມາຊິກ, ແຕ້ມ ແລະ Coupon

  • ກົດສ່ວນ ລູກຄ້າ / ສະມາຊິກ ເພື່ອຄົ້ນຫາສະມາຊິກ.
  • ຖ້າບໍ່ພົບສະມາຊິກ ສາມາດເພີ່ມໃໝ່ຈາກ modal ເລືອກສະມາຊິກໄດ້ທັນທີ.
  • ຖ້າເປີດໃຊ້ Loyalty ລະບົບຈະສະສົມແຕ້ມ ແລະ ໃຫ້ໃຊ້ແຕ້ມຫັກຍອດຕາມເງື່ອນໄຂ.
  • Coupon ໃຊ້ໂດຍປ້ອນລະຫັດ ແລ້ວກົດໃຊ້. ລະບົບຈະກວດກັບໂປຣໂມຊັ່ນທີ່ເປີດໃຊ້.

ເມນູແລະປຸ່ມຫຼັກໃນໜ້າ POS

ເມນູ / ປຸ່ມວິທີໃຊ້ງານຜົນທີ່ໄດ້
ຊ່ອງສະແກນ / ຄົ້ນຫາສະແກນ barcode ຫຼື ພິມລະຫັດ/ຊື່ສິນຄ້າ.ສິນຄ້າຖືກເພີ່ມເຂົ້າບິນ ຫຼື ສະແດງຜົນຄົ້ນຫາ.
Catalogກົດເມື່ອຕ້ອງການເລືອກສິນຄ້າດ້ວຍມື, ເລືອກໝວດໝູ່ ແລະກົດສິນຄ້າ.ສິນຄ້າຖືກເພີ່ມເຂົ້າບິນ.
ໂຫຼດໂປຣໂມຊັ່ນໃໝ່ກົດເມື່ອ Admin ແກ້ໂປຣໂມຊັ່ນ ແລະຕ້ອງການໃຊ້ທັນທີ.ສ່ວນຫຼຸດ / coupon ຖືກອັບເດດໃນ POS.
ຈໍລູກຄ້າກົດເພື່ອເປີດຈໍທີ່ 2 ສຳລັບລູກຄ້າເບິ່ງລາຍການແລະຍອດຊຳລະ.ໜ້າ Customer Display ຖືກເປີດ ແລະ sync ກັບບິນປັດຈຸບັນ.
ສະຫຼຸບຂາຍ / ສົ່ງເງິນກົດເພື່ອເບິ່ງຍອດຂາຍມື້ນີ້ ແລະບັນທຶກເງິນທີ່ສົ່ງ.ລາຍການສົ່ງເງິນຖືກບັນທຶກໃນລະບົບ.
ແຈ້ງເຕືອນໜີ້ລູກຄ້າກົດເພື່ອເບິ່ງບິນ credit ທີ່ຄົບກຳນົດ ຫຼື ເກີນກຳນົດ.ພະນັກງານຮູ້ລາຍການທີ່ຄວນຕິດຕາມຮັບຊຳລະ.
ຮັບຄືນກົດເພື່ອຄົ້ນບິນ, ເລືອກສິນຄ້າທີ່ຮັບຄືນ ແລະບັນທຶກການຄືນເງິນ.ລະບົບສ້າງ return record ແລະປັບຂໍ້ມູນຕາມການຮັບຄືນ.
ປະຫວັດບິນກົດເພື່ອເບິ່ງບິນຂາຍຫຼ້າສຸດ, ພິມຊ້ຳ, ຮັບຄືນ ຫຼື ຍົກເລີກບິນຕາມສິດ.ກວດສອບບິນໄດ້ຈາກໜ້າ POS ໂດຍບໍ່ຕ້ອງເຂົ້າ Admin.

3. ການຊຳລະເງິນ ແລະ ອອກໃບບິນ

ວິທີຊຳລະ

4. ພັກບິນ, ຮັບຄືນ ແລະ ສົ່ງເງິນ

ພັກບິນ

ການພັກບິນໃຊ້ເມື່ອລູກຄ້າຍັງບໍ່ພ້ອມຈ່າຍ ແຕ່ຕ້ອງການຮັບລູກຄ້າຄົນຕໍ່ໄປ. ກົດ ພັກບິນ, ຕັ້ງຊື່ອ້າງອີງ ແລ້ວລະບົບຈະເກັບບິນໄວ້. ເມື່ອຕ້ອງການກັບມາຂາຍຕໍ່ ໃຫ້ເປີດລາຍການບິນພັກ ແລະ ກົດໂຫຼດ.

ຮັບຄືນ / ຄືນເງິນ

  1. ກົດ ຮັບຄືນ ຈາກໜ້າ POS ຫຼື ຈາກປະຫວັດການຂາຍ.
  2. ປ້ອນເລກບິນ ຫຼື ຄົ້ນຫາບິນທີ່ຕ້ອງການ.
  3. ເລືອກຈຳນວນສິນຄ້າທີ່ຮັບຄືນ ໂດຍບໍ່ເກີນຈຳນວນທີ່ຄືນໄດ້.
  4. ເລືອກວິທີຄືນເງິນ ແລະ ບັນທຶກເຫດຜົນ.
  5. ກົດ ບັນທຶກຮັບຄືນ. ລະບົບຈະບັນທຶກປະຫວັດ ແລະ ປັບສະຕັອກຕາມຂໍ້ມູນ.

ສະຫຼຸບການຂາຍ ແລະ ສົ່ງເງິນ

ພະນັກງານສາມາດເປີດສະຫຼຸບການຂາຍປະຈຳວັນເພື່ອເບິ່ງຍອດຂາຍ, ຍອດເງິນສົດ, ແລະ ບັນທຶກການສົ່ງເງິນ. ໃຫ້ກວດເງິນສົດຈິງ, ກົດໃສ່ຈຳນວນທີ່ສົ່ງ, ລະບຸຜູ້ຮັບ ແລະ ບັນທຶກໝາຍເຫດຖ້າມີສ່ວນຕ່າງ.

5. ຄູ່ມືທຸກເມນູໃນ Admin Console

ສ່ວນນີ້ອະທິບາຍທຸກເມນູທີ່ຢູ່ໃນ Sidebar ຂອງ Admin Console. ຖ້າຜູ້ໃຊ້ບາງຄົນບໍ່ເຫັນເມນູໃດໜຶ່ງ, ໃຫ້ Admin ກວດສິດໃນເມນູ User ແລະ ສິດ.

ພາບລວມ

ເມນູຂັ້ນຕອນໃຊ້ງານຜົນທີ່ຄວນກວດ
ແຜງຄວບຄຸມເຂົ້າ Admin ແລ້ວເລືອກເມນູນີ້. ກວດຍອດຂາຍມື້ນີ້, ອາທິດ, ເດືອນ, ກຳໄລ, stock ແລະການເງິນ. ກົດໂຫຼດໃໝ່ຖ້າຕ້ອງການຂໍ້ມູນຫຼ້າສຸດ.ຕົວເລກຂາຍ, ໜີ້, stock ແລະການເງິນຄວນກົງກັບການດຳເນີນງານຈິງ.

ສິນຄ້າ

ເມນູຂັ້ນຕອນໃຊ້ງານຜົນທີ່ຄວນກວດ
ຈັດການສິນຄ້າກົດເພີ່ມ ຫຼື ແກ້ໄຂສິນຄ້າ, ປ້ອນລະຫັດ, barcode, ຊື່, ໝວດໝູ່, ຍີ່ຫໍ້, ຫົວໜ່ວຍ, ຕົ້ນທຶນ, ລາຄາຂາຍ, stock ແລະຮູບສິນຄ້າ. ຖ້າມີຫຼາຍຮຸ່ນໃຫ້ເຂົ້າ Variants.Barcode ບໍ່ຊ້ຳ, ລາຄາຖືກ, stock ຖືກ, ແລະສິນຄ້າຄົ້ນເຫັນໃນ POS.
ໝວດໝູ່ / ຍີ່ຫໍ້ / ຫົວໜ່ວຍເພີ່ມຫຼືແກ້ໄຂຊື່ category, brand ແລະ unit ທີ່ໃຊ້ໃນຟອມສິນຄ້າ.ຊື່ບໍ່ຊ້ຳ, ສະກົດຖືກ, ແລະເລືອກໄດ້ໃນການເພີ່ມສິນຄ້າ.
ຜູ້ສະໜອງເພີ່ມ supplier, ຂໍ້ມູນຕິດຕໍ່, ທີ່ຢູ່ ແລະບັນທຶກປະຫວັດການຕິດຕໍ່. ໃຊ້ສຳລັບການຊື້ເຂົ້າແລະໜີ້ supplier.Supplier ປາກົດໃນໜ້າຊື້ເຂົ້າ ແລະຂໍ້ມູນຕິດຕໍ່ຄົບ.
ສິນຄ້າຄວນສັ່ງເພີ່ມເຂົ້າເບິ່ງລາຍການ stock ຕ່ຳ, ກວດຈຸດ reorder ແລະນຳໄປສ້າງໃບສະເໜີຊື້ ຫຼື ການຊື້.ສິນຄ້າທີ່ຕ່ຳກວ່າຈຸດກຳນົດຖືກຕິດຕາມກ່ອນຂາດ stock.
ພິມປ້າຍລາຄາເລືອກສິນຄ້າ, ຈຳນວນປ້າຍ, ຮູບແບບປ້າຍ ແລະກົດພິມ.ປ້າຍລາຄາ/Barcode ອ່ານງ່າຍ ແລະລາຄາກົງກັບ POS.
ປັບປຸງສະຕັອກເລືອກສິນຄ້າ, ລະບຸຈຳນວນປັບເພີ່ມ/ຫຼຸດ, ສາເຫດ ແລະບັນທຶກ.Stock ປ່ຽນຕາມຈຳນວນທີ່ບັນທຶກ ແລະມີເຫດຜົນໃນປະຫວັດ.
ນັບສິນຄ້າ (Stock take)ສ້າງຮອບນັບ, ເລືອກສິນຄ້າ/ສາຂາ, ປ້ອນຈຳນວນທີ່ນັບໄດ້, ກວດສ່ວນຕ່າງ ແລະຢືນຢັນ.ລະບົບສະແດງສ່ວນຕ່າງລະຫວ່າງ stock ລະບົບແລະຂອງຈິງ.
ໂອນຣະຫວ່າງສາຂາກົດສ້າງໃບໂອນ, ເລືອກສາຂາຕົ້ນທາງ/ປາຍທາງ, ເພີ່ມສິນຄ້າ, ລະບຸຈຳນວນ, ບັນທຶກ ແລະຢືນຢັນການຮັບ.Stock ຕົ້ນທາງຫຼຸດ ແລະປາຍທາງເພີ່ມເມື່ອສຳເລັດ.

ຊື້

ເມນູຂັ້ນຕອນໃຊ້ງານຜົນທີ່ຄວນກວດ
ໃບສະເໜີຊື້ສ້າງໃບຂໍຊື້, ເລືອກ supplier ຫຼືສິນຄ້າ, ລະບຸຈຳນວນ, ໝາຍເຫດ ແລະບັນທຶກ. ເມື່ອອະນຸມັດແລ້ວສາມາດແປງເປັນການຊື້.ລາຍການຂໍຊື້ມີສະຖານະຖືກ ແລະຈຳນວນກົງກັບຄວາມຕ້ອງການ.
ລະບົບຊື້ເຂົ້າກົດສ້າງການຊື້, ເລືອກ supplier, ເພີ່ມສິນຄ້າ, ລະບຸຕົ້ນທຶນ, ຈຳນວນ, ວິທີຈ່າຍ, ວັນຄົບກຳນົດຖ້າເປັນໜີ້, ແລະບັນທຶກ.Stock ເພີ່ມ, ຕົ້ນທຶນອັບເດດ, ແລະຖ້າເປັນໜີ້ຈະໄປຢູ່ເມນູໜີ້ supplier.
ໜີ້ຜູ້ສະໜອງຄົ້ນບິນຊື້ທີ່ຍັງຄ້າງ, ກວດຍອດຄົງເຫຼືອ, ວັນຄົບກຳນົດ ແລະປະຫວັດການຈ່າຍ.ສະຖານະ paid/partial/pending ຖືກຕ້ອງ ແລະຍອດຄ້າງກົງ.
ຊຳລະໃຫ້ເຈົ້າໜີ້ເລືອກ supplier ຫຼືບິນທີ່ຈະຈ່າຍ, ໃສ່ຈຳນວນ, ວິທີຈ່າຍ, ເລກອ້າງອີງ ແລະບັນທຶກ.ຍອດໜີ້ supplier ຫຼຸດລົງ ແລະມີຫຼັກຖານການຈ່າຍ.

ຂາຍ

ເມນູຂັ້ນຕອນໃຊ້ງານຜົນທີ່ຄວນກວດ
ໃບສະເໜີລາຄາກົດສ້າງໃບສະເໜີ, ເລືອກລູກຄ້າ, ເພີ່ມສິນຄ້າ, ກຳນົດລາຄາ/ສ່ວນຫຼຸດ, ບັນທຶກ ແລະພິມ. ຖ້າລູກຄ້າຕົກລົງ ໃຫ້ແປງເປັນບິນຂາຍ.ລາຄາ, ວັນໝົດອາຍຸ ແລະຂໍ້ມູນລູກຄ້າກົງກັນ.
Layby / ມັດຈຳສ້າງ layby, ເລືອກລູກຄ້າ, ສິນຄ້າ, ຍອດມັດຈຳ, ວັນຮັບສິນຄ້າ, ແລະບັນທຶກງວດຈ່າຍເມື່ອລູກຄ້າຈ່າຍເພີ່ມ.ສະຖານະມັດຈຳ, ຍອດຄ້າງ ແລະສິນຄ້າຈອງຖືກຕ້ອງ.
ປະຫວັດການຂາຍຄົ້ນຫາຕາມເລກບິນ, ລູກຄ້າ, ວັນທີ ຫຼືວິທີຈ່າຍ. ເປີດລາຍລະອຽດ, ພິມຊ້ຳ, ຮັບຄືນ ຫຼືຍົກເລີກຕາມສິດ.ບິນຂາຍທຸກໃບສາມາດກວດກັບເງິນແລະ stock ໄດ້.
ຮັບຄືນ / ຄືນເງິນເລືອກບິນ, ເລືອກສິນຄ້າທີ່ຄືນໄດ້, ລະບຸຈຳນວນ, ວິທີຄືນເງິນ ແລະສາເຫດ.ບໍ່ຮັບຄືນເກີນຈຳນວນທີ່ຂາຍ ແລະມີເລກ return ໃນປະຫວັດ.
ລາຍການສົ່ງເງິນກວດກາລາຍການສົ່ງເງິນຈາກ POS, ກວດຜູ້ສົ່ງ, ຜູ້ຮັບ, ຍອດຄາດໝາຍ, ຍອດຈິງ ແລະສ່ວນຕ່າງ.ຍອດເງິນສົດມື້ນັ້ນກົງກັບເງິນຈິງທີ່ຮັບມອບ.

ການເງິນ

ເມນູຂັ້ນຕອນໃຊ້ງານຜົນທີ່ຄວນກວດ
ບັນທຶກລາຍຮັບເພີ່ມລາຍຮັບອື່ນທີ່ບໍ່ແມ່ນບິນຂາຍ, ລະບຸປະເພດ, ຈຳນວນ, ວັນທີ ແລະໝາຍເຫດ.ລາຍຮັບຖືກບວກໃນ cash flow ແລະລາຍງານ.
ບັນທຶກລາຍຈ່າຍເພີ່ມລາຍຈ່າຍຮ້ານ, ເຊັ່ນ ຄ່າຂົນສົ່ງ, ຄ່າໄຟ, ຄ່າອຸປະກອນ, ພ້ອມໝາຍເຫດ.ລາຍຈ່າຍຖືກຫັກໃນ cash flow ແລະກວດຍ້ອນຫຼັງໄດ້.
ການເຄື່ອນໄຫວເງິນສົດເລືອກຊ່ວງວັນທີ ເພື່ອກວດລາຍຮັບ, ລາຍຈ່າຍ, ຍອດຂາຍເງິນສົດ ແລະຍອດສຸດທິ.ເງິນສົດເຂົ້າ-ອອກກົງກັບບັນທຶກແລະການສົ່ງເງິນ.
ລາຍງານພາສີ / VATເລືອກຊ່ວງວັນທີ, ກວດຍອດຂາຍກ່ອນ VAT, VAT ແລະຍອດລວມ.ຍອດ VAT ກົງກັບການຕັ້ງຄ່າ VAT ຂອງບໍລິສັດ.
ກຳໄລ / COGSເລືອກຊ່ວງລາຍງານ ເພື່ອກວດລາຍຮັບ, ຕົ້ນທຶນສິນຄ້າຂາຍ, ກຳໄລ ແລະ margin.ຕົ້ນທຶນຄວນຖືກຕັ້ງຄ່າໃນສິນຄ້າ ເພື່ອໃຫ້ລາຍງານແມ່ນຍຳ.
KPI ພະນັກງານເລືອກຊ່ວງວັນທີ ແລະກວດຍອດຂາຍ, ຈຳນວນບິນ, ມູນຄ່າສະເລ່ຍ ແລະຜົນງານຕໍ່ພະນັກງານ.ໃຊ້ປຽບທຽບຜົນງານ, ວາງແຜນກະງານ ແລະຝຶກອົບຮົມ.

ລູກຄ້າ

ເມນູຂັ້ນຕອນໃຊ້ງານຜົນທີ່ຄວນກວດ
ສະມາຊິກເພີ່ມ/ແກ້ໄຂຊື່, ເບີໂທ, ທີ່ຢູ່, ຂໍ້ມູນສະສົມແຕ້ມ ແລະເບິ່ງປະຫວັດການຊື້.ສະມາຊິກຄົ້ນໄດ້ໃນ POS ແລະຂໍ້ມູນຕິດຕໍ່ຖືກ.
ໜີ້ລູກຄ້າກວດບິນຂາຍຕິດໜີ້, ຍອດຄ້າງ, ວັນຄົບກຳນົດ, ສະຖານະ overdue ແລະປະຫວັດຈ່າຍ.ບິນທີ່ຄວນຕິດຕາມຖືກເຫັນຊັດ ແລະຍອດຄ້າງຖືກ.
ຊຳລະຈາກລູກໜີ້ເລືອກບິນ/ລູກຄ້າ, ປ້ອນຈຳນວນທີ່ຮັບ, ວິທີຮັບເງິນ, ເລກອ້າງອີງ ແລະບັນທຶກ.ຍອດຫນີ້ລູກຄ້າຫຼຸດລົງ ແລະມີປະຫວັດການຮັບຊຳລະ.

ຕັ້ງຄ່າ

ເມນູຂັ້ນຕອນໃຊ້ງານຜົນທີ່ຄວນກວດ
User ແລະ ສິດສ້າງຜູ້ໃຊ້, ຕັ້ງ username/password, role, ແລະເລືອກສິດ Access, Edit, Delete ຕາມເມນູ.ຜູ້ໃຊ້ເຫັນສະເພາະເມນູທີ່ໄດ້ຮັບສິດ ແລະບໍ່ມີສິດເກີນຈຳເປັນ.
ປະຫວັດການເຮັດວຽກເລືອກຊ່ວງວັນທີ ຫຼືຄົ້ນຕາມຜູ້ໃຊ້/ການກະທຳ ເພື່ອກວດການແກ້ໄຂ, ລຶບ, ສ້າງ ຫຼືການບັນທຶກສຳຄັນ.ການກະທຳສຳຄັນມີຜູ້ເຮັດ, ເວລາ ແລະລາຍລະອຽດໃຫ້ກວດສອບ.
ຄູ່ມືການໃຊ້ງານເຂົ້າເມນູນີ້ເພື່ອເບິ່ງ preview PDF, ກົດເປີດ PDF ຫຼືດາວໂຫຼດເອກະສານ.ຜູ້ໃຊ້ສາມາດອ່ານຄູ່ມືໄດ້ຈາກໃນລະບົບໂດຍກົງ.
ສຳຮອງ / ສົ່ງອອກຂໍ້ມູນກົດດາວໂຫຼດ backup JSON ທັງລະບົບ ຫຼື Export CSV ແຕ່ລະຕາຕະລາງ.ໄຟລ໌ backup ຖືກເກັບໄວ້ບ່ອນປອດໄພ ແລະທົດສອບວ່າເປີດໄດ້.
ກຳນົດລາຄາຂາຍຄົ້ນສິນຄ້າ, ແກ້ຕົ້ນທຶນ/ລາຄາຂາຍ ຫຼື bulk update, ກວດປະຫວັດລາຄາ ແລະບັນທຶກ.ລາຄາໃໝ່ສະແດງໃນ POS ແລະມີ history ການປ່ຽນແປງ.
ໂປຣໂມຊັ່ນສ້າງ promotion, ເລືອກປະເພດສ່ວນຫຼຸດ/coupon/ຂອງແຖມ, ກຳນົດຊ່ວງວັນທີ, ເງື່ອນໄຂ ແລະ active.POS ຄຳນວນໂປຣໂມຊັ່ນຖືກ ແລະ promotion ໝົດອາຍຸບໍ່ຖືກນຳໃຊ້.
ຕັ້ງຄ່າແຕ້ມສະສົມເປີດ/ປິດ loyalty, ກຳນົດຍອດເງິນຕໍ່ແຕ້ມ, ມູນຄ່າແຕ້ມ, ຂັ້ນຕ່ຳໃນການແລກ ແລະບັນທຶກ.ແຕ້ມສະສົມ/ແລກແຕ້ມໃນ POS ຄຳນວນຕາມເງື່ອນໄຂ.
ສະກຸນເງິນ / ອັດຕາເພີ່ມຫຼືແກ້ສະກຸນເງິນ, symbol, exchange rate, ການເປີດໃຊ້ ແລະລຳດັບ.ໜ້າຊຳລະສະແດງສະກຸນເງິນທີ່ active ແລະຄຳນວນເປັນ LAK ຖືກ.
ສາຂາເພີ່ມສາຂາ, ລະຫັດ, ສະຖານະ active, ສາຂາຫຼັກ ແລະລຳດັບສະແດງ.POS ແລະ stock ເລືອກສາຂາໄດ້ຖືກຕ້ອງ.
ແຂວງ / ເມືອງ / ບ້ານຈັດການຂໍ້ມູນທີ່ຢູ່ທີ່ໃຊ້ໃນຟອມສະມາຊິກ ແລະຂໍ້ມູນລູກຄ້າ.ຕົວເລືອກທີ່ຢູ່ຖືກ, ບໍ່ຊ້ຳ ແລະຄົ້ນຫາໄດ້.
ຂໍ້ມູນບໍລິສັດຕັ້ງຊື່ຮ້ານ, logo, slogan, ທີ່ຢູ່, ເບີໂທ, VAT, ການປັດເສດ ແລະຂໍ້ຄວາມໃນໃບບິນ.Login, POS, Admin ແລະໃບບິນສະແດງຂໍ້ມູນຮ້ານຖືກຕ້ອງ.
ຮູບແບບເລກບິນກຳນົດ format ເລກບິນຂາຍ, ເລກຈ່າຍ, PO ແລະເອກະສານອື່ນ ໂດຍໃຊ້ຕົວແທນປີ/ເດືອນ/ລຳດັບ.ເລກເອກະສານບໍ່ຊ້ຳ ແລະອ່ານງ່າຍຕາມນະໂຍບາຍຮ້ານ.

6. ລາຍລະອຽດການໃຊ້ແຕ່ລະເມນູ

ບົດນີ້ແຍກອະທິບາຍເປັນເມນູຕໍ່ເມນູ. ໃຊ້ສ່ວນນີ້ເປັນຄູ່ມືສອນພະນັກງານໃໝ່ ຫຼື ກວດກາວ່າຜູ້ໃຊ້ເຂົ້າໃຈວິທີເຮັດວຽກຫຼັກຂອງແຕ່ລະໜ້າແລ້ວ.

1. ແຜງຄວບຄຸມ

ເສັ້ນທາງ: Admin Console → ພາບລວມ → ແຜງຄວບຄຸມ
  • ໃຊ້ເບິ່ງຍອດຂາຍມື້ນີ້, ອາທິດນີ້, ເດືອນນີ້, ກຳໄລ, ຕົ້ນທຶນ, ການຮັບຄືນ, ໜີ້ລູກຄ້າ ແລະ ໜີ້ supplier.
  • ກ່ອນປິດຮ້ານ ໃຫ້ກວດຍອດຂາຍມື້ນີ້ກັບໜ້າສະຫຼຸບ POS ແລະ ລາຍການສົ່ງເງິນ.
  • ຖ້າຕົວເລກບໍ່ອັບເດດ ໃຫ້ກົດໂຫຼດໃໝ່ ຫຼື refresh browser.

2. ຈັດການສິນຄ້າ

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ຈັດການສິນຄ້າ
  • ເພີ່ມສິນຄ້າໂດຍປ້ອນລະຫັດສິນຄ້າ, barcode, ຊື່, ໝວດໝູ່, ຍີ່ຫໍ້, ຫົວໜ່ວຍ, ຕົ້ນທຶນ, ລາຄາຂາຍ ແລະ stock ເລີ່ມຕົ້ນ.
  • ຖ້າສິນຄ້າມີຫຼາຍຂະໜາດ/ສີ/ຮຸ່ນ ໃຫ້ໃຊ້ໜ້າ variants ເພື່ອແຍກ barcode ແລະລາຄາຂອງແຕ່ລະຮຸ່ນ.
  • ຫຼັງບັນທຶກ ທົດສອບຄົ້ນຫາສິນຄ້າໃນ POS ແລະສະແກນ barcode ເພື່ອຢືນຢັນ.

3. ໝວດໝູ່ / ຍີ່ຫໍ້ / ຫົວໜ່ວຍ

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ໝວດໝູ່ / ຍີ່ຫໍ້ / ຫົວໜ່ວຍ
  • ໃຊ້ສ້າງຂໍ້ມູນພື້ນຖານທີ່ຈະນຳໄປເລືອກໃນຟອມສິນຄ້າ.
  • ກ່ອນເພີ່ມຊື່ໃໝ່ ໃຫ້ຄົ້ນຫາກ່ອນເພື່ອຫຼີກລ້ຽງຊື່ຊ້ຳ.
  • ຖ້າລຶບຂໍ້ມູນທີ່ສິນຄ້າໃຊ້ຢູ່ ຄວນກວດສິນຄ້າທີ່ຜູກຢູ່ກ່ອນ.

4. ຜູ້ສະໜອງ

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ຜູ້ສະໜອງ
  • ເພີ່ມຊື່ supplier, ເບີໂທ, ທີ່ຢູ່, ຜູ້ຕິດຕໍ່ ແລະໝາຍເຫດການຊື້ຂາຍ.
  • ໃຊ້ຂໍ້ມູນນີ້ໃນໜ້າຊື້ເຂົ້າ, ໃບສະເໜີຊື້ ແລະ ໜີ້ຜູ້ສະໜອງ.
  • ຄວນບັນທຶກປະຫວັດການຕິດຕໍ່ເມື່ອມີການເຈລະຈາລາຄາ ຫຼື ເງື່ອນໄຂຈ່າຍເງິນ.

5. ສິນຄ້າຄວນສັ່ງເພີ່ມ

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ສິນຄ້າຄວນສັ່ງເພີ່ມ
  • ເບິ່ງສິນຄ້າທີ່ stock ຕ່ຳກວ່າຈຸດສັ່ງຊື້ ຫຼືມີຄວາມສ່ຽງຂາດສິນຄ້າ.
  • ກວດຍອດຂາຍຫຼ້າສຸດ ແລະ supplier ກ່ອນສ້າງໃບສະເໜີຊື້.
  • ຖ້າສິນຄ້າບໍ່ຄວນສັ່ງ ໃຫ້ປັບ reorder level ໃນຂໍ້ມູນສິນຄ້າ.

6. ພິມປ້າຍລາຄາ

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ພິມປ້າຍລາຄາ
  • ເລືອກສິນຄ້າທີ່ຈະພິມ, ກຳນົດຈຳນວນປ້າຍ ແລະຮູບແບບທີ່ຈະໃຊ້.
  • ກ່ອນພິມຈຳນວນຫຼາຍ ໃຫ້ທົດສອບພິມ 1 ໜ້າ ເພື່ອກວດ barcode ແລະຂະໜາດ.
  • ຫຼັງຕິດປ້າຍ ໃຫ້ສະແກນຈາກ POS ເພື່ອກວດວ່າ barcode ຖືກຕ້ອງ.

7. ປັບປຸງສະຕັອກ

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ປັບປຸງສະຕັອກ
  • ໃຊ້ເມື່ອ stock ຈິງບໍ່ກົງກັບລະບົບ ເຊັ່ນ ສິນຄ້າເສຍ, ນັບຜິດ, ສູນເສຍ ຫຼື ພົບເກີນ.
  • ລະບຸສາຂາ, ສິນຄ້າ, ຈຳນວນທີ່ປັບ ແລະເຫດຜົນໃຫ້ຊັດ.
  • ຄວນໃຫ້ຫົວໜ້າກວດກ່ອນປັບ stock ຈຳນວນຫຼາຍ.

8. ນັບສິນຄ້າ (Stock take)

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ນັບສິນຄ້າ (Stock take)
  • ສ້າງຮອບນັບ stock, ເລືອກສາຂາ ແລະລາຍການສິນຄ້າທີ່ຈະນັບ.
  • ປ້ອນຈຳນວນທີ່ນັບໄດ້ຈິງ, ກວດສ່ວນຕ່າງ ແລະບັນທຶກຄຳອະທິບາຍສຳລັບລາຍການຜິດປົກກະຕິ.
  • ຢືນຢັນຜົນນັບເມື່ອກວດຄົບແລ້ວ ເພື່ອໃຫ້ stock ໃນລະບົບຖືກປັບຕາມ.

9. ໂອນຣະຫວ່າງສາຂາ

ເສັ້ນທາງ: Admin Console → ສິນຄ້າ → ໂອນຣະຫວ່າງສາຂາ
  • ສ້າງໃບໂອນ, ເລືອກສາຂາຕົ້ນທາງ ແລະສາຂາປາຍທາງ.
  • ເພີ່ມສິນຄ້າ, ກຳນົດຈຳນວນ, ກວດ stock ທີ່ຕົ້ນທາງ ແລະບັນທຶກໃບໂອນ.
  • ເມື່ອສາຂາປາຍທາງຮັບຂອງ ໃຫ້ຢືນຢັນໃບໂອນ ເພື່ອໃຫ້ stock ປາຍທາງເພີ່ມ.

10. ໃບສະເໜີຊື້

ເສັ້ນທາງ: Admin Console → ຊື້ → ໃບສະເໜີຊື້
  • ໃຊ້ສ້າງຄຳຂໍຊື້ກ່ອນຈະບັນທຶກເປັນການຊື້ເຂົ້າຈິງ.
  • ເພີ່ມສິນຄ້າ, ຈຳນວນ, supplier ແລະເຫດຜົນການຂໍຊື້.
  • ຫຼັງອະນຸມັດ ສາມາດແປງເປັນລາຍການຊື້ເຂົ້າໄດ້ ເພື່ອຫຼຸດການປ້ອນຂໍ້ມູນຊ້ຳ.

11. ລະບົບຊື້ເຂົ້າ

ເສັ້ນທາງ: Admin Console → ຊື້ → ລະບົບຊື້ເຂົ້າ
  • ເລືອກ supplier, ເພີ່ມສິນຄ້າ, ລະບຸຈຳນວນ, ລາຄາຊື້, ສະກຸນເງິນ ແລະວິທີຈ່າຍ.
  • ຖ້າຊື້ແບບຕິດໜີ້ ໃຫ້ກຳນົດວັນຄົບກຳນົດ ແລະຍອດທີ່ຈ່າຍແລ້ວ.
  • ຫຼັງບັນທຶກ stock ຈະເພີ່ມ ແລະຂໍ້ມູນຕົ້ນທຶນຈະຖືກໃຊ້ໃນລາຍງານກຳໄລ/COGS.

12. ໜີ້ຜູ້ສະໜອງ

ເສັ້ນທາງ: Admin Console → ຊື້ → ໜີ້ຜູ້ສະໜອງ
  • ໃຊ້ກວດລາຍການຊື້ທີ່ຍັງຄ້າງຈ່າຍໃຫ້ supplier.
  • ກວດສະຖານະຈ່າຍ, ຍອດລວມ, ຍອດຈ່າຍແລ້ວ, ຍອດຄ້າງ ແລະວັນຄົບກຳນົດ.
  • ໃຊ້ເປັນລາຍການຕິດຕາມກ່ອນຈ່າຍເງິນໃຫ້ supplier.

13. ຊຳລະໃຫ້ເຈົ້າໜີ້

ເສັ້ນທາງ: Admin Console → ຊື້ → ຊຳລະໃຫ້ເຈົ້າໜີ້
  • ເລືອກບິນຊື້ຫຼື supplier ທີ່ຈະຈ່າຍ, ລະບຸຈຳນວນເງິນ ແລະວິທີຈ່າຍ.
  • ບັນທຶກເລກອ້າງອີງ ຫຼືແນບຫຼັກຖານຖ້າມີ.
  • ຫຼັງບັນທຶກ ກວດວ່າຍອດຄ້າງ supplier ຫຼຸດລົງຖືກຕ້ອງ.

14. ໃບສະເໜີລາຄາ

ເສັ້ນທາງ: Admin Console → ຂາຍ → ໃບສະເໜີລາຄາ
  • ສ້າງ quotation ໃຫ້ລູກຄ້າກ່ອນຂາຍຈິງ, ເລືອກລູກຄ້າ ແລະເພີ່ມສິນຄ້າ.
  • ປັບລາຄາ, ສ່ວນຫຼຸດ, ວັນໝົດອາຍຸ ແລະໝາຍເຫດຂໍ້ຕົກລົງ.
  • ເມື່ອລູກຄ້າຕົກລົງ ສາມາດແປງ quotation ເປັນບິນຂາຍ.

15. Layby / ມັດຈຳ

ເສັ້ນທາງ: Admin Console → ຂາຍ → Layby / ມັດຈຳ
  • ໃຊ້ຈອງສິນຄ້າໃຫ້ລູກຄ້າໂດຍຮັບເງິນມັດຈຳກ່ອນ.
  • ບັນທຶກລູກຄ້າ, ສິນຄ້າ, ຍອດມັດຈຳ, ວັນຄົບກຳນົດ ແລະປະຫວັດຈ່າຍເພີ່ມ.
  • ເມື່ອຈ່າຍຄົບ ໃຫ້ complete layby ເພື່ອປິດລາຍການ.

16. ປະຫວັດການຂາຍ

ເສັ້ນທາງ: Admin Console → ຂາຍ → ປະຫວັດການຂາຍ
  • ຄົ້ນຫາບິນຂາຍຕາມເລກບິນ, ວັນທີ, ລູກຄ້າ, ພະນັກງານ ຫຼືວິທີຈ່າຍ.
  • ໃຊ້ກວດລາຍລະອຽດບິນ, ພິມຊ້ຳ, ຮັບຄືນ ຫຼືຍົກເລີກບິນຕາມສິດ.
  • ຖ້າບິນຖືກຍົກເລີກ ຄວນກວດ stock ແລະລາຍງານເງິນວ່າຖືກປັບແລ້ວ.

17. ຮັບຄືນ / ຄືນເງິນ

ເສັ້ນທາງ: Admin Console → ຂາຍ → ຮັບຄືນ / ຄືນເງິນ
  • ເລືອກບິນຂາຍ, ລາຍການສິນຄ້າ ແລະຈຳນວນທີ່ຈະຮັບຄືນ.
  • ລະບຸວິທີຄືນເງິນ ແລະເຫດຜົນການຮັບຄືນໃຫ້ຊັດເຈນ.
  • ກວດວ່າຈຳນວນທີ່ຮັບຄືນບໍ່ເກີນຈຳນວນທີ່ຂາຍ ແລະມີເລກ return ໃນປະຫວັດ.

18. ລາຍການສົ່ງເງິນ

ເສັ້ນທາງ: Admin Console → ຂາຍ → ລາຍການສົ່ງເງິນ
  • ໃຊ້ກວດລາຍການເງິນສົດທີ່ cashier ບັນທຶກສົ່ງຈາກໜ້າ POS.
  • ກວດຍອດທີ່ຄວນສົ່ງ, ຍອດຈິງ, ສ່ວນຕ່າງ, ຜູ້ສົ່ງ, ຜູ້ຮັບ ແລະໝາຍເຫດ.
  • ຖ້າຍອດບໍ່ກົງ ຄວນກວດປະຫວັດບິນ, ການຮັບຄືນ ແລະລາຍຈ່າຍທີ່ບັນທຶກໃນມື້ນັ້ນ.

19. ບັນທຶກລາຍຮັບ

ເສັ້ນທາງ: Admin Console → ການເງິນ → ບັນທຶກລາຍຮັບ
  • ໃຊ້ບັນທຶກເງິນເຂົ້າທີ່ບໍ່ແມ່ນຈາກການຂາຍ POS ເຊັ່ນ ເງິນຄືນ, ລາຍຮັບອື່ນ ຫຼືປັບຍອດ.
  • ລະບຸປະເພດ, ຈຳນວນ, ວັນທີ, ຜູ້ຮັບ ແລະໝາຍເຫດ.
  • ກວດວ່າລາຍຮັບປາກົດໃນ cash flow ຂອງຊ່ວງວັນທີນັ້ນ.

20. ບັນທຶກລາຍຈ່າຍ

ເສັ້ນທາງ: Admin Console → ການເງິນ → ບັນທຶກລາຍຈ່າຍ
  • ໃຊ້ບັນທຶກຄ່າໃຊ້ຈ່າຍຮ້ານ ເຊັ່ນ ຄ່າຂົນສົ່ງ, ຄ່າຊ່ອມ, ຄ່າອຸປະກອນ ຫຼືຄ່າບໍລິການ.
  • ລະບຸປະເພດ, ຈຳນວນ, ວັນທີ, ຜູ້ຈ່າຍ ແລະໝາຍເຫດ/ເລກອ້າງອີງ.
  • ກ່ອນລາຍງານກຳໄລ ຄວນກວດວ່າລາຍຈ່າຍຖືກຈັດປະເພດຖືກ.

21. ການເຄື່ອນໄຫວເງິນສົດ

ເສັ້ນທາງ: Admin Console → ການເງິນ → ການເຄື່ອນໄຫວເງິນສົດ
  • ໃຊ້ກວດເງິນສົດເຂົ້າ-ອອກຈາກການຂາຍ, ລາຍຮັບອື່ນ ແລະລາຍຈ່າຍ.
  • ເລືອກຊ່ວງວັນທີ ແລ້ວກວດຍອດສຸດທິກັບເງິນຈິງ.
  • ຖ້າມີສ່ວນຕ່າງ ໃຫ້ກວດບິນຍົກເລີກ, ຮັບຄືນ ແລະລາຍການສົ່ງເງິນ.

22. ລາຍງານພາສີ / VAT

ເສັ້ນທາງ: Admin Console → ການເງິນ → ລາຍງານພາສີ / VAT
  • ເລືອກຊ່ວງວັນທີເພື່ອກວດຍອດຂາຍກ່ອນ VAT, ຍອດ VAT ແລະຍອດລວມ.
  • ກວດອັດຕາ VAT ແລະວິທີຄຳນວນກັບການຕັ້ງຄ່າບໍລິສັດ.
  • ໃຊ້ລາຍງານນີ້ສຳລັບກວດສອບກ່ອນສົ່ງຂໍ້ມູນພາສີ.

23. ກຳໄລ / COGS

ເສັ້ນທາງ: Admin Console → ການເງິນ → ກຳໄລ / COGS
  • ໃຊ້ວິເຄາະລາຍຮັບ, ຕົ້ນທຶນສິນຄ້າຂາຍ, ກຳໄລຂັ້ນຕົ້ນ ແລະ margin.
  • ຕົ້ນທຶນຈະແມ່ນຍຳເມື່ອຂໍ້ມູນຕົ້ນທຶນໃນສິນຄ້າ ແລະການຊື້ເຂົ້າຖືກຕ້ອງ.
  • ໃຊ້ລາຍງານນີ້ເພື່ອປັບລາຄາຂາຍ ຫຼືກວດສິນຄ້າທີ່ກຳໄລຕ່ຳ.

24. KPI ພະນັກງານ

ເສັ້ນທາງ: Admin Console → ການເງິນ → KPI ພະນັກງານ
  • ເລືອກຊ່ວງວັນທີ ແລະກວດຍອດຂາຍ, ຈຳນວນບິນ, ຍອດສະເລ່ຍຕໍ່ບິນ ແລະຜົນງານຕາມ cashier.
  • ໃຊ້ເພື່ອປຽບທຽບຜົນງານ, ວາງແຜນກະ ແລະກຳນົດການຝຶກອົບຮົມ.
  • ຄວນກວດວ່າ cashier login ດ້ວຍບັນຊີຂອງຕົນເອງ ເພື່ອໃຫ້ KPI ຖືກ.

25. ສະມາຊິກ

ເສັ້ນທາງ: Admin Console → ລູກຄ້າ → ສະມາຊິກ
  • ເພີ່ມຊື່ລູກຄ້າ, ເບີໂທ, ທີ່ຢູ່, ລະຫັດສະມາຊິກ ແລະຂໍ້ມູນຕິດຕໍ່.
  • ເບິ່ງປະຫວັດການຊື້, ແຕ້ມສະສົມ ແລະລະດັບສະມາຊິກ.
  • ກ່ອນສ້າງສະມາຊິກໃໝ່ ຄວນຄົ້ນຫາເບີໂທກ່ອນເພື່ອຫຼີກການຊ້ຳ.

26. ໜີ້ລູກຄ້າ

ເສັ້ນທາງ: Admin Console → ລູກຄ້າ → ໜີ້ລູກຄ້າ
  • ກວດບິນຂາຍຕິດໜີ້, ຍອດຄ້າງ, ວັນຄົບກຳນົດ ແລະສະຖານະເກີນກຳນົດ.
  • ໃຊ້ຕິດຕາມລູກຄ້າທີ່ຕ້ອງໂທຫາ ຫຼືແຈ້ງເຕືອນຮັບຊຳລະ.
  • ກວດປະຫວັດການຈ່າຍກ່ອນບັນທຶກຍອດຮັບໃໝ່.

27. ຊຳລະຈາກລູກໜີ້

ເສັ້ນທາງ: Admin Console → ລູກຄ້າ → ຊຳລະຈາກລູກໜີ້
  • ເລືອກບິນ credit ຫຼືລູກຄ້າທີ່ຈະຮັບຊຳລະ.
  • ປ້ອນຈຳນວນ, ວິທີຮັບເງິນ, ເລກອ້າງອີງ ແລະໝາຍເຫດ.
  • ຫຼັງບັນທຶກ ກວດວ່າຍອດຄ້າງຂອງລູກຄ້າຫຼຸດລົງຖືກຕ້ອງ.

28. User ແລະ ສິດ
